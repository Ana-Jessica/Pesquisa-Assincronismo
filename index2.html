1. Assincronismo

    2. Definição: Em programação, assincronismo refere-se à capacidade de um programa executar múltiplas tarefas de forma concorrente, sem que uma tarefa precise esperar a conclusão da outra. Isso permite que o programa continue a responder a outras solicitações enquanto uma tarefa está em andamento, como por exemplo, uma operação de entrada/saída (I/O), como a leitura de um arquivo ou a realização de uma requisição HTTP.

    1. AJAX (Asynchronous Javascript And XML)
      • AJAX permite que páginas da web sejam atualizadas assincronamente, trocando dados com o servidor sem recarregar a página inteira.
    2. Promises
      • A Promises fornece uma maneira de lidar com operações assíncronas, retornando um objeto que representa o eventual sucesso ou falha de uma operação assíncrona.
    3. Fetch API
      • A Fetch API é uma interface mais simples para fazer requisições assíncronas, retornando uma Promise.
    4. Async / Await
      • asynce e await forneça uma maneira mais clara de trabalhar com promessas, tornando o código mais legível e semelhante ao código síncrono.
    5. Quais vantagens/desvantagens de usar cada uma das opções acima?
      6. Técnica Vantagens Desvantagens
      AJAX Compatível com a maioria dos navegadores Sintaxe mais detalhada e mais propensa a erros
      Promises Simples de encadear várias operações assíncronas Código pode ficar aninhado em callbacks
      Fetch API Código mais limpo e legível do XMLHttpRequest Falta suporte para interceptar certas requisições nativamente
      Async / Await Mais fácil de ler e escrever, especialmente com operações longas Requer cuidado com erros, pois pode gerar blocos try/catch
  2. Outros tópicos:
    1. Hoisting
      • Hoisting é o comportamento do JavaScript onde variáveis ​​e declarações de funções são "elevadas" ao topo de seu escopo antes da execução.
    2. Arrow Functions (especialmente diferenças com relação as funções normais)
      • As funções de seta têm uma sintaxe mais curta e não são próprias this ou arguments.
    3. Desestruturação (Destructuring)
      • A desestruturação permite extrair valores de arrays ou propriedades de objetos em variáveis ​​distintas.
    4. Closure
      • Um closure é uma função que lembra o escopo onde foi criado, mesmo depois de esse escopo ter sido finalizado.
